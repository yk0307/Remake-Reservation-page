## 目的
- /hub/系ハブページのUIを最小工数で引き締まった印象へ刷新する。
- 既存のリンク機能は維持し、スタイルとレイアウトのみ調整する。

## スコープ
- public/app.js のマークアップ生成ロジック
- public/styles.css のスタイル定義とテーマ変数
- .agents/tasks/session.md へのログ追記

## 背景
- 既存UIはボタン幅や余白の統一感が弱く、背景がフラットで“しっかり感”が不足している。
- 安全領域やフォーカスリングの扱いを揃え、アクセシビリティ指標の向上が求められている。

## 方針
- CSSカスタムプロパティで指定されたトークン値を表現し、既存の設定ロジック(applyTheme)に追従させる。
- マークアップを`<main>`配下のセクションに組み替え、ボタン要素にラベルとアイコンのフレックス配置を導入する。
- PC幅では2カラムのレスポンシブグリッドを適用し、モバイルは縦並びのままとする。

## タスク
1. 現状のHTML生成とCSSトークンを洗い出し、追加すべきクラス/変数を整理する。
2. public/app.js のテンプレートを更新し、新しいクラス名と内部構造（ラベル・アイコン）を組み込む。
3. public/styles.css に新しいレイアウト/ボタンスタイル/背景グラデーション/フォーカスリングを実装し、アクセシビリティ要件を満たす。
4. npm run lint / npm run test を実行し、結果をsession.mdに記録する。

## 懸念・リスク
- SPA構造のため他ページとスタイルを共有している可能性があり、副作用が出ないようクラススコープをハブ向けに限定する必要がある。
- fetchで取得するテーマ変数が不足している場合はCSS側で安全なデフォルトを定義してカバーする。

## 完了条件
- モバイル/デスクトップ双方で指定のレイアウト・ホバー・フォーカス挙動が再現される。
- Lighthouse（ローカル想定）アクセシビリティ95以上を再現可能な設計になっている。
- lint / test が成功し、session.md に最新ログが追加されている。
